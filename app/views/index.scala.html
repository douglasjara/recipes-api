Welcome to Recipes REST API!

This project is an example of play framework use, which manage recipes with their ingredients and suggestions. Additional information of each recipe like price, calories, difficulty level, and number of guests are included too.

Please follow the directions in the README.md file in the top-level project directory to run the project.

Features

This project is an example of play framework use, wich contains the following features:

  • REST resources design (routes)
  • Operation resources design (verbs)
  • Models: attributes, finder, etc
  • 1 to 1 model relation
  • 1 to N model relation
  • N to M model relation (adapation, see References in README.md)
  • Model validations
  • Custom validations
  • Controllers
  • JSON and XML views
  • i18n (internationalization): English and Spanish
  • Cache (only for fun, 5 seconds in GET methods. POST and PUT remove cache.)
  • Action composition

DB config

Default DB for use is postgreSQL, but you can re define environmental variables to set your favorite database:

  • DB_DRIVER
  • DB_URL
  • DB_USERNAME
  • DB_PASSWORD

This project supports the following DBMS:

  • h2 database
  • PostgreSQL
  • MySQL
